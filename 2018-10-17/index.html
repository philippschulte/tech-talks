Semantic Commits

A pattern for more readable messages

Philipp Schulte - AppEng Summit 2018

What are semantic commit messages?

Precise rules over how a git commit messages can be formatted. Usually you see this pattern only for packages, libraries, and languages.

Motivation

Pattern

type ( scope ) :   Short description

Types

feat, fix, docs, style, refactor, perf, test, build, ci, chore, revert

Types, Scope, WTF

Global Setup

  1. Install commitizen and an adapter globally
    2. 
							$ npm install -g commitizen cz-conventional-changelog
  2. Create a .czrc file in your home directory, with path referring to the preferred, globally installed, commitizen adapter
    3. 
								$ echo '{ "path": "cz-conventional-changelog" }' > ~/.czrc

Git Workflow


						$ git add .
						$ git cz
						$ git push

Note: You can still use all the git commit options with git cz, for example: git cz -a.

Conclusion

  • Simple and fast global setup
  • Easy usage via commitizen's command prompt
  • Instead of git commit we use git cz
  • No disadvantages because all options are still available
  • Readable commits
  • Consistent format and history
  • Easy filtering e.g. git log --grep="feat"
